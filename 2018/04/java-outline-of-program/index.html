Java 程序设计概述

发表于 2018-04-25 | 分类于 读书笔记 |

前言

因为最近的几件事让我意识到自己的基础实在是太差，便买了《Java 核心技术·卷Ⅰ（第10版）》来强化自己的基础。为了能加深印象，所以打算边读边写，将书中的内容经过理解转变为自己的知识。

关于Java

20世纪90年代，硬件领域出现了单片式计算机系统，Sun公司为了抢占市场先机，在1991年成立了一个称为Green的项目小组。由于考虑到当时C++所具有的优势，该项目组的研究人员首先考虑采用C++来编写程序。但对于硬件资源极其匮乏的单片式系统来说，C++程序过于复杂和庞大。所以他们开始着眼于新语言的开发，他们对C++进行改造，成功开发了一种名为Oak的面向对象的语言，经过几年的发展，Oak改名为Java，并成为当时各大公司竞相购买的目标。

从1996年Java 第一次发布，便引起了人们极大兴趣。在《Java 核心技术·卷一（第1版）》中有这样关于Java 的描述：

作为一种计算机语言，Java 的广告词确实有点夸大其词。然而，Java 的确是一种优秀的程序设计语言。作为一个名副其实的程序设计人员，使用Java 无疑是一个好的选择。有人认为：Java 将有望成为一种最优秀的程序设计语言，但还需要一个相当长的发展时期。一旦一种语言应用于某个领域，与现存代码的相容性问题就摆在了人们面前。

Java 有许多非常优秀的语言特性，同时，相容性这个问题导致Java 并不是十分完美。

Java 其实是一个完整的平台，有庞大的库在使Java 变得功能更加齐全，更加出色。也正是因为库的存在，对广大的程序设计人员有着不可抗拒的吸引力。

Java 语言特性

简单性

Java 是一门面向对象编程语言，不仅吸收了C++语言的各种优点，还摒弃了C++里难以理解的多继承、指针等概念。同时Java 的目标之一就是支持开发能够在小型机器上独立运行的软件。所以才有了较小类库的、适用于嵌入式设备的Java 微型版（Java Micro Edition）

面向对象

在Java 这门语言中，流传着这样一句话：“万物皆对象”。的确，我们周边的所有事物，看得见的，看不见的都能被抽象成为一个对象。相比C++中的对象，Java 以简单的接口概念替代了难以理解的多重继承，这使得在用Java 编程时，代码将会更便于理解。

分布式

由于Java 有其庞大的库在支撑，其中就包含有处理像HTTP/FTP之类的TCP/IP协议的库，使得Java 程序能同过URL打开或者访问网络上的对象。

健壮性

Java 投入了大量精力进行早期的问题检测与运行时检测，显出易出错的情况。同时Java 编译器能够检测许多在其他语言中仅在运行时才能够检测出来的问题。

安全性

Java 安全模型基于一个可定制的“沙盒”，Java 软件程序可在其中安全运行，对系统或用户无潜在风险。无论Java 代码来自哪里，都不可能脱离“沙盒”独立运行。
Java 的安全性体现在：

  • 强大的数据类型管理
  • 自动内存管理
  • 字节码验证
  • 安全的类加载

体系结构中立

在运行Java 文件之前，编译器会为Java 文件生成一中特殊格式的文件（ .class 文件）。这是一种经过编译的文件，与计算机体系结构完全无关的字节码文件，它可以很轻易的在任何机器上解释并运行，而且还可以动态的翻译成本地机器代码。

可移植性

  • 固定大小的数据类型
  • 固定格式存储与传输二进制数据（消除字节顺序的错误）
  • 采用标准的Unicode 格式存储
  • 部分可移植的系统接口（给出不同平台下的实现）
  • 可移植的Java API

解释型

在这里，我们先谈谈两个概念，编译型语言与解释型语言：

  • 编译型语言：把做好的源程序全部编译成二进制代码的可运行程序。然后，可直接运行这个程序。
  • 解释型语言：把做好的源程序翻译一句，然后执行一句，直至结束。

它们又有何区别？

  • 编译型语言：执行速度快、效率高；依赖编译器、跨平台性差些。如C、C++、Delphi、Pascal，Fortran。
  • 解释型语言：执行速度慢、效率低；依赖解释器、跨平台性好。如Java、Basic。

通俗的讲，编译语言是在编译后可以直接运行，而解释语言的执行需要一个解释环境。

Java 其实很特殊，Java 程序也需要编译，但是没有直接编译称为机器语言，而是编译称为字节码，然后用解释的方式来执行字节码。所以Java 属于解释型语言。

高性能

Java 编译生成的字节码文件可以在运行时刻动态的翻译成对应运行这个应用的特定CPU的机器码，大大提高了性能。

多线程

在现在这个科技高速发展的时代，摩尔定律其实已经接近失效。如今，并发性逐步成为一个评定系统好坏的标准。为了解决并发性的问题，人们引入了多线程这一概念.Java 是第一个支持并发程序设计的主流语言。

动态性

Java 库中可以在不对客户端造成任何影响的前提下，动态的添加新方法或者实例变量。
